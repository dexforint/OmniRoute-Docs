# Auto-Combo: позвольте OmniRoute выбирать лучший ИИ за вас

> **TL;DR**: Установите модель `auto`, и OmniRoute будет автоматически выбирать лучшего ИИ-провайдера для каждого запроса. Настройка не требуется.

---

## Что это делает

Вместо выбора конкретной ИИ-модели (например, GPT-4o или Claude) вы можете позволить OmniRoute **автоматически выбирать лучшую** для каждого запроса. Учитываются:

- **Здоровье** — работает ли провайдер прямо сейчас?
- **Скорость** — насколько он быстрый?
- **Стоимость** — сколько он стоит?
- **Качество** — хорош ли он в этом типе задач?
- **Ёмкость** — осталась ли у него квота?

OmniRoute оценивает всех подключённых провайдеров и выбирает лучшего. В случае сбоя автоматически пробует следующего.

---

## Быстрый старт

**Шаг 1**: Установите модель `auto` в вашей IDE или CLI:

```
model: "auto"
```

**Шаг 2**: Всё! Остальное OmniRoute берёт на себя.

**Шаг 3** (необязательно): Используйте вариант для конкретных задач:

```
model: "auto/coding"    # Лучший для кода
model: "auto/fast"      # Самый быстрый ответ
model: "auto/cheap"     # Самый дешёвый вариант
```

---

## Какой «auto» выбрать?

| Если вам нужно... | Используйте | Лучше всего для | Как это работает |
|-------------------|-------------|-----------------|------------------|
| **Лучшее в целом** | `auto` | Общие вопросы, чат | Баланс скорости, стоимости и качества |
| **Лучший код** | `auto/coding` | Написание кода, отладка | Выбирает модели, сильные в задачах кода |
| **Самый быстрый ответ** | `auto/fast` | Быстрые ответы, низкая задержка | Скорость важнее всего |
| **Самый дешёвый вариант** | `auto/cheap` | Экономия денег | Выбирает самого дешёвого провайдера |
| **Самая умная модель** | `auto/smart` | Сложные задачи | Качество прежде всего + исследует новые модели |
| **Максимальная доступность** | `auto/offline` | Когда провайдеры заняты | Выбирает провайдеров с наибольшей ёмкостью |

### Примеры

```bash
# Общий чат — сбалансированный
curl http://localhost:20128/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[{"role":"user","content":"Hello!"}]}'

# Генерация кода — качество прежде всего
curl http://localhost:20128/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"auto/coding","messages":[{"role":"user","content":"Write a Python function"}]}'

# Быстрый ответ — скорость прежде всего
curl http://localhost:20128/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"auto/fast","messages":[{"role":"user","content":"What is 2+2?"}]}'
```

## Как это работает (простое объяснение)

Когда вы отправляете запрос с `model: "auto"`, OmniRoute:

1. **Смотрит на всех подключённых провайдеров** — каждый добавленный вами провайдер (OpenAI, Anthropic, Google и т.д.)
2. **Оценивает каждого** по 5 факторам:
   - Работает ли он? (здоровье)
   - Есть ли ёмкость? (квота)
   - Сколько он стоит? (цена)
   - Насколько он быстрый? (скорость)
   - Хорош ли он в этой задаче? (качество)
3. **Выбирает лучшего** — провайдер с наивысшей оценкой получает ваш запрос
4. **Самовосстановление** — в случае сбоя OmniRoute автоматически пробует следующего

### Система оценки

Каждый провайдер получает оценку от 0 до 1. Чем выше оценка, тем лучше соответствие.

| Фактор | Вес | Что это значит |
|--------|-----|----------------|
| Здоровье | 20% | Работает ли провайдер? (состояние circuit breaker) |
| Квота | 15% | Осталась ли ёмкость? |
| Стоимость | 15% | Насколько он дорогой? (дешевле = выше оценка) |
| Скорость | 12% | Насколько он быстрый? (ниже задержка = выше оценка) |
| Соответствие задаче | 8% | Хорош ли он в этом типе задач? |
| Стабильность | 5% | Стабилен ли он? (низкая частота ошибок) |
| Уровень | 5% | Уровень аккаунта (Ultra > Pro > Free) |
| Прочее | 20% | Контекстное сходство, плотность соединений и т.д. |

### Как варианты меняют оценку

Каждый вариант использует свои веса:

| Вариант | Приоритет | Ключевые веса |
|---------|-----------|---------------|
| `auto` | Баланс | health=20%, quota=15%, cost=15% |
| `auto/coding` | Качество | taskFit=37%, stability=15% |
| `auto/fast` | Скорость | latency=32%, health=28% |
| `auto/cheap` | Стоимость | cost=37% |
| `auto/smart` | Качество + исследование | taskFit=37%, exploration=10% |
| `auto/offline` | Ёмкость | quota=37%, health=28% |

## Как обрабатываются сбои

У OmniRoute **три уровня защиты**:

### 1. Автоматическое переключение (fallback)
Если лучший провайдер даёт сбой, OmniRoute автоматически пробует следующего. От вас ничего не требуется.

### 2. Самовосстановление
Если провайдер продолжает давать сбои:
- **Оценка < 0.2** → исключается на 5 минут
- **Circuit breaker открыт** → автоматически исключается
- **Более 50% провайдеров недоступны** → режим инцидента (без исследования)

### 3. Аварийное переключение
Если все провайдеры отказали, OmniRoute в крайнем случае направляет запросы на стабильных бесплатных провайдеров (например, Kiro или Qoder).

---

## Поддержка нескольких аккаунтов

Если у вас несколько аккаунтов одного провайдера (например, два ключа OpenAI), OmniRoute рассматривает каждый как **отдельного кандидата**. Это значит:

- У аккаунта A осталась квота → используем его
- Аккаунт B ограничен по скорости → пропускаем
- Аккаунт C дешевле → предпочитаем его

Каждый аккаунт оценивается независимо по собственным здоровью, квоте и скорости.

---

## Исследование по принципу «бандита»

OmniRoute периодически **исследует** новых провайдеров, чтобы обнаруживать лучшие варианты:

- **По умолчанию**: 5% запросов уходят случайным провайдерам
- **Auto/smart**: 10% запросов на исследование
- **Отключается**, когда более 50% провайдеров нездоровы

Это помогает OmniRoute узнавать, какие провайдеры лучше подходят под ваши паттерны использования.

## Частые вопросы

### «Он всегда будет выбирать самую дорогую модель?»

**Нет.** Стоимость по умолчанию составляет лишь 15% оценки. Дешёвый, быстрый и здоровый провайдер может обойти дорогого. Используйте `auto/cheap`, если хотите ещё сильнее приоритизировать стоимость.

### «Что, если провайдер выйдет из строя?»

OmniRoute автоматически пропускает его и пробует следующего. Если провайдер продолжает давать сбои, он временно исключается (5–30 минут). От вас ничего не требуется.

### «Можно ли увидеть, какой провайдер был использован?»

Проверьте заголовки ответа — OmniRoute включает провайдера и модель, использованные в каждом ответе.

### «Он учится на моём использовании?»

Да! Система оценки использует исторические данные (задержка, частота ошибок, доля успешных запросов), чтобы со временем принимать лучшие решения.

### «В чём разница между `auto` и `auto/smart`?»

- `auto` — сбалансированный, 5% исследования
- `auto/smart` — качество прежде всего (те же веса, что у `auto/coding`), 10% исследования

Используйте `auto/smart`, когда нужно максимальное качество и вы не против периодического исследования.

### «Можно ли принудительно указать конкретного провайдера?»

Да! Используйте комбо со стратегией `priority` вместо `auto`. Подробности — в [техническом справочнике](../routing/AUTO-COMBO.md).

### «Чем это отличается от round-robin?»

Round-robin перебирает провайдеров по очереди. Auto-combo **оценивает каждого провайдера** и выбирает лучшего. Это умнее — учитываются здоровье, скорость, стоимость и качество.
