# Языковые пакеты сжатия

Сжатие Caveman может загружать языко-специфичные пакеты правил в дополнение к встроенным английским правилам.
Это сохраняет ядро движка стабильным, позволяя пакетам для португальского, испанского, немецкого, французского, японского и
будущих языков развиваться независимо.

## Расположение

Языковые пакеты находятся в:

```txt
open-sse/services/compression/rules/<language>/
```

Текущие поставляемые пакеты (проверены по содержимому каталога `rules/`):

| Язык                | Каталог        | Присутствующие категории правил                     |
| ------------------- | -------------- | --------------------------------------------------- |
| Английский          | `rules/en/`    | `context`, `dedup`, `filler`, `structural`, `ultra` |
| Испанский           | `rules/es/`    | `context`, `dedup`, `filler`, `structural`, `ultra` |
| Португальский (Бразилия) | `rules/pt-BR/` | `context`, `dedup`, `filler`, `structural`, `ultra` |
| Индонезийский       | `rules/id/`    | `context`, `dedup`, `filler`, `structural`, `ultra` |
| Немецкий            | `rules/de/`    | `context`, `filler`, `structural`                   |
| Французский         | `rules/fr/`    | `context`, `filler`, `structural`                   |
| Японский            | `rules/ja/`    | `context`, `filler`, `structural`                   |

> **Замечание о паритете:** пакеты `en`, `es`, `pt-BR` и `id` имеют все 5 категорий; `de`, `fr`, `ja` поставляются с 3 категориями. Отсутствующие категории `dedup` и `ultra` молча откатываются на английские встроенные правила. Приветствуются вклады, добавляющие `dedup.json` и `ultra.json` для меньших пакетов.
>
> Пакет `pt-BR` основан на **[Troglodita](https://github.com/leninejunior/troglodita)** от Lenine Júnior — системе сжатия, спроектированной с нуля для грамматики бразильского португальского (сокращение плеоназмов, удаление слов-паразитов PT-BR, технические сокращения для бразильского dev-сообщества).
>
> Канонический список категорий и схема по категориям находятся в [`open-sse/services/compression/rules/_schema.json`](../../open-sse/services/compression/rules/_schema.json) (JSON Schema draft 2020-12).

## Определение языка

`languageDetector.ts` использует лёгкие эвристики для вывода языка из текста промпта.
Настроенный язык по умолчанию по-прежнему учитывается, а определение можно отключить в конфигурации, когда
требуется точный контроль.

Вывод определения используется только для выбора пакетов правил. Он не меняет маршрутизацию по провайдерам,
выбор локали или язык UI.

## Форма конфигурации

Настройки сжатия могут включать:

```json
{
  "languageConfig": {
    "enabled": true,
    "defaultLanguage": "en",
    "autoDetect": true,
    "enabledPacks": ["en", "pt-BR", "es", "id", "de", "fr", "ja"]
  },
  "cavemanConfig": {
    "language": "en",
    "autoDetectLanguage": true,
    "enabledLanguagePacks": ["en", "pt-BR", "es", "id", "de", "fr", "ja"]
  }
}
```

`languageConfig` управляет значениями по умолчанию для панели/предпросмотра. `cavemanConfig` — конфигурация движка времени выполнения,
используемая, когда Caveman сжимает текст сообщений.

## Добавление языкового пакета

1. Создайте `open-sse/services/compression/rules/<language>/<pack>.json`.
2. Используйте формат правила Caveman из `docs/compression/COMPRESSION_RULES_FORMAT.md`.
3. Сохраняйте замены консервативными и избегайте изменения кода, идентификаторов, URL или JSON.
4. Добавьте или обновите тесты на выбор языка и поведение замен.
5. Раскройте новые метки панели/i18n, если язык появляется в селекторах UI.

## API

Доступные пакеты можно запросить:

```bash
curl http://localhost:20128/api/compression/language-packs
```

Эндпоинт предпросмотра принимает переопределения языковой конфигурации:

```bash
curl -X POST http://localhost:20128/api/compression/preview \
  -H "Content-Type: application/json" \
  -d '{
    "mode": "standard",
    "text": "Por favor, eu gostaria que voce basicamente resumisse isso.",
    "config": {
      "languageConfig": {
        "defaultLanguage": "pt-BR",
        "autoDetect": true
      }
    }
  }'
```

## SHARED_BOUNDARIES (v3.8.0)

Все 6 языковых пакетов получили положение `SHARED_BOUNDARIES` в v3.8.0, применяемое на каждой
интенсивности Caveman (LITE, FULL, ULTRA). Оно предписывает движку сохранять эти паттерны дословно,
независимо от удаления окружающих слов-паразитов:

| Тип паттерна                     | Пример                                 |
| -------------------------------- | -------------------------------------- |
| Огороженные блоки кода           | ` ```python\n...\n``` `                |
| Встроенный код                   | `` `my_var` ``                         |
| URL                              | `https://example.com/path`             |
| Пути файлов (абсолютные + относительные) | `/etc/hosts`, `./src/index.ts` |
| Заголовки ошибок                 | `Error:`, `TypeError:`, `SyntaxError:` |
| Строки трассировки стека         | `  at functionName (file.ts:12:3)`     |

Эти паттерны заполняют `DEFAULT_CAVEMAN_CONFIG.preservePatterns` (ранее `[]`).
Константа находится в `open-sse/services/compression/types.ts`.

### Почему это важно

Без SHARED_BOUNDARIES агрессивные режимы Caveman могли бы вырезать содержимое, выглядевшее как повторяющаяся
проза, но на самом деле бывшее фрагментом кода, путём файла или стеком ошибки. SHARED_BOUNDARIES действует как
языконезависимая страховочная сеть, применяемая до запуска правил удаления паразитов.

### Настройка preservePatterns

Дополнительные паттерны можно добавить во время выполнения через настройки сжатия:

````json
{
  "cavemanConfig": {
    "preservePatterns": [
      "```[\\s\\S]*?```",
      "`[^`]+`",
      "https?://\\S+",
      "(?:/|\\./)[^\\s]+",
      "\\b(?:Error|TypeError|SyntaxError|RangeError):",
      "\\s+at\\s+\\S+\\s+\\(\\S+:\\d+:\\d+\\)"
    ]
  }
}
````

Пользовательские паттерны расширяют (не заменяют) 6 значений по умолчанию.

## Эксплуатационные замечания

- Английские встроенные правила остаются фолбэком, когда языковой пакет отсутствует.
- Невалидные встроенные JSON-пакеты проваливают валидацию, чтобы релизные ресурсы молча не деградировали.
- Пакеты правил содержат только данные и не должны импортировать код или выполнять произвольную логику.
- Слой аналитики сжатия записывает выбранный режим и движок, но не полный текст промпта.
