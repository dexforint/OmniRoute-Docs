# API бесплатных прокси

OmniRoute поставляет курируемый пул бесплатных прокси в таблице `free_proxies`,
синхронизируемый с внешними провайдерами (1proxy, proxifly, iplocate, webshare).
Панель управления показывает их в **Settings → Free Proxies**. Этот документ
охватывает серверную фильтрацию, сортировку, подсчёт и отчётность об ошибках синхронизации,
которые экспонирует маршрут списка.

## Маршрут списка — `GET /api/settings/free-proxies`

Возвращает отфильтрованный, отсортированный, пагинированный срез плюс общее число. Фильтрация и
подсчёт происходят в SQL, поэтому UI может показать реальный total (например, `Total: 0`)
без загрузки каждой строки в память.

### Параметры запроса

| Параметр          | Тип                                | Default   | Значение                                                                                         |
| ----------------- | ---------------------------------- | --------- | ---------------------------------------------------------------------------------------------- |
| `search`          | string                             | `""`      | Чувствительный к регистру `LIKE` по колонке host (и source).                                   |
| `protocol`        | string                             | `""`      | Фильтр `type`: `http` / `https` / `socks4` / `socks5`. Пусто = все.                             |
| `country`         | string                             | `""`      | Фильтр `countryCode` (ISO-2). Пусто = все.                                                     |
| `minQuality`      | number                             | `0`       | Только строки с `qualityScore >= minQuality`. `0` = без порога.                                |
| `disabledSources` | string                             | `""`      | Разделённый запятыми список id источников для исключения (например, `proxifly,webshare`).      |
| `sortBy`          | `quality` \| `latency` \| `recent` | `quality` | `quality` = score по убыванию; `latency` = latency по возрастанию (nulls last); `recent` = `lastValidated` по убыванию. |
| `offset`          | number                             | `0`       | Начало пагинации.                                                                              |
| `limit`           | number                             | `50`      | Размер страницы (ограничивается на сервере).                                                   |

### Ответ

```json
{
  "success": true,
  "data": {
    "proxies": [/* FreeProxyRecord[] */],
    "total": 137,
    "hasMore": true
  },
  "stats": {
    "total": 137,
    "inPool": 12,
    "avgQuality": 64.2,
    "bySource": [{ "source": "1proxy", "count": 90 }],
    "lastSyncAt": "2026-07-11T09:30:00.000Z"
  },
  "syncErrors": {
    "proxifly": ["HTTP 429 from upstream"],
    "webshare": ["network timeout"]
  }
}
```

`total` отражает отфильтрованный тотал **до** пагинации, поэтому UI может рендерить
`Total: N` и `hasMore` независимо. `syncErrors` ключуется по id источника и
заполняется только для источников, которые провалили последнюю синхронизацию — результат `Total: 0`
никогда не бывает тихим.

## Добавление в пул — `POST /api/settings/free-proxies/[id]/add-to-pool`

Продвигает бесплатный прокси в управляемый пул `proxy_registry`. Валидирует
апстрим сначала; при успехе возвращает новый id прокси пула и измеренную задержку.

## Синхронизация — `POST /api/settings/free-proxies/sync`

Перетягивает все включённые источники (или подмножество в `{ "sources": [...] }`). Каждый
источник синхронизируется независимо; падающий источник записывается в `syncErrors` и
остальные всё равно завершаются, поэтому частичные синхронизации никогда не стирают предыдущие хорошие данные.

## Статистика — `GET /api/settings/free-proxies/stats`

Возвращает агрегат `total / inPool / avgQuality / bySource / lastSyncAt`
без полезной нагрузки строк — используется виджетами заголовка панели управления.
