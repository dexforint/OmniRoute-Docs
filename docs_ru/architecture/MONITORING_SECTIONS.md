# Мониторинг и стоимость — структура навигации

> Реализовано в Group B (plan 16). См. `src/shared/constants/sidebarVisibility.ts`.

---

## Навигация верхнего уровня

Сайдбар панели (после Group B) имеет следующие разделы верхнего уровня по порядку:

```
Home
Providers
Combos
API Keys
Settings
Analytics
Costs          ← НОВОЕ (Group B, plan 16)
Monitoring     ← РЕОРГАНИЗОВАНО (Group B, plan 16)
...
```

---

## Раздел Costs (новый, уровень 1)

Префикс пути: `/dashboard/costs/`

| Элемент       | URL                                  | Описание                                      |
| ------------- | ------------------------------------ | --------------------------------------------- |
| Overview      | `/dashboard/costs`                   | Агрегированная панель стоимости (перенесено из Analytics) |
| Pricing       | `/dashboard/costs/pricing`           | Таблица цен по моделям                        |
| Budget        | `/dashboard/costs/budget`            | Пороги бюджета + алерты                       |
| Quota Sharing | `/dashboard/costs/quota-share`       | Пулы Quota Share + использование              |
| Plan Config   | `/dashboard/costs/quota-share/plans` | Переопределения планов по провайдерам         |

**Обоснование**: Pricing, Budget и Quota Sharing ранее находились под
`Monitoring > Costs Parameters`. Перенос их в отдельный раздел верхнего уровня
делает их обнаруживаемыми без навигации через инструменты наблюдаемости.

## Раздел Monitoring (реорганизован)

Раздел Monitoring теперь имеет **Activity наверху**, за которым следуют **3 подгруппы**:

```
Monitoring
├── Activity             ← Лента событий (элемент верхнего уровня)
├── Logs group
│   ├── Logs (all)
│   ├── Proxy Logs
│   └── Console Logs
├── Audit group
│   ├── Audit Log
│   ├── MCP Audit
│   └── A2A Audit
└── System group
    ├── Health
    └── Runtime
```

### Что изменилось по сравнению со старой структурой

| До                                                                          | После                                             |
| --------------------------------------------------------------------------- | ------------------------------------------------- |
| Activity = вкладка внутри Logs, рендерившая Audit Log                       | Activity = отдельная лента (`/dashboard/activity`) |
| Группа Costs Parameters в Monitoring                                         | Перенесена в раздел Costs                         |
| Плоский список: Logs, Activity (logs), Audit, Health, Runtime, Pricing, Budget, Quota | Структурированная 3-группа + отдельный раздел Costs |

## Activity vs Audit Log

Теперь это два разных понятия:

| Измерение        | Activity (`/dashboard/activity`)                       | Audit Log (`/dashboard/audit`)            |
| ---------------- | ------------------------------------------------------ | ----------------------------------------- |
| **Назначение**   | Пользовательская лента событий ("что произошло недавно") | Лог соответствия / безопасности           |
| **Источник данных** | `GET /api/compliance/audit-log?level=high`          | `GET /api/compliance/audit-log?level=all` |
| **Формат**       | Таймлайн, сгруппированный по дням, человекочитаемые глаголы + иконки | Плотная пагинированная таблица, 50/стр |
| **Фильтры**      | Категория типа события                                 | Действие, серьёзность, актёр, диапазон дат |
| **Экспорт**      | Недоступен                                             | JSON экспорт                               |
| **Фильтр по актёру** | Неприменимо                                        | Фильтруется по актёру                      |
| **Показанные события** | Только высокоуровневые действия (allowlist)      | Все события аудита                         |

### Allowlist высокоуровневых действий

Определён в `src/lib/audit/highLevelActions.ts`. Контролирует, какие события появляются в
ленте Activity. Allowlist включает:

- События добавления/удаления/тестирования провайдеров
- Создание/обновление/удаление Combo
- Жизненный цикл API-ключей (создание, отзыв, ротация)
- Достижение порога бюджета
- Вход/выход аутентификации
- Создание сессии cloud-агента
- Регистрация MCP-инструмента
- Создание/удаление Webhook
- Изменения пулов/планов квот (`quota.*` действия, Group B)
- События платформы (обновление, деплой)
- Установка/удаление Skill

События, не входящие в этот список, появляются только в Audit Log.

### Добавление нового высокоуровневого действия

Отредактируйте `src/lib/audit/highLevelActions.ts` и добавьте строку действия в
`HIGH_LEVEL_ACTIONS`. Для этого требуется PR (список — это код, а не конфигурируемый через БД).
Соответствующую иконку можно добавить в `src/lib/audit/activityIcons.ts`.

## Редирект: `/dashboard/logs/activity`

Старый путь `/dashboard/logs/activity` постоянно перенаправляется (HTTP 308) на
`/dashboard/activity` через `permanentRedirect()` в
`src/app/(dashboard)/dashboard/logs/activity/page.tsx`.

Устаревший ID сайдбара `logs-activity` сохраняется в `HIDEABLE_SIDEBAR_ITEM_IDS`
(но удалён из `SIDEBAR_DEFINITIONS`), чтобы не ломать пресеты пользователей, которые
ссылаются на старый ID.

## i18n

Пространства имён, добавленные Group B:

| Ключ пространства имён      | Покрывает                                                        |
| --------------------------- | ---------------------------------------------------------------- |
| `sidebar.costsSection`      | Метка раздела Costs                                              |
| `sidebar.activity`          | Элемент сайдбара Activity                                        |
| `sidebar.logsGroup`         | Метка подгруппы Logs                                             |
| `sidebar.systemGroup`       | Метка подгруппы System                                           |
| `sidebar.costsOverview`     | Элемент обзора Costs                                             |
| `activity.*`                | Все строки страницы Activity (заголовок, глаголы, фильтры, пустое состояние) |

Языки-источники истины: `pt-BR` и `en`. Все остальные 39 локалей откатываются на
английский через механизм fallback `next-intl` (настроен в `src/i18n/config.ts`).
