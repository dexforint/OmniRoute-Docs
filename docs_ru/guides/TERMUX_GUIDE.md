# Headless-установка Termux

OmniRoute может работать как headless-сервер на Android через Termux. Десктопное приложение Electron в Termux не поддерживается, но веб-панель и OpenAI-совместимый API доступны из локального браузера или с других устройств в той же сети.

## Предварительные требования

Установите Termux из F-Droid или релизов GitHub, затем обновите пакеты и установите инструменты сборки, необходимые для нативных зависимостей, таких как `better-sqlite3`.

```bash
pkg update
pkg upgrade
pkg install nodejs python build-essential git
```

> **Версия Node.js:** OmniRoute требует Node `>=22.22.2 <23 || >=24.0.0 <27` (соответствует `engines` в `package.json` / `SUPPORTED_NODE_RANGE`). Пакет `nodejs-lts` в Termux обычно поставляет Node 20 LTS, который **больше не поддерживается** — установите `pkg install nodejs` (текущий) и проверьте, что `node --version` показывает ветку 22.x/24.x+.

Если компиляция нативных пакетов завершилась с ошибкой, повторите команду `pkg install` выше, а затем повторите установку OmniRoute.

## Установка

Запустите последний опубликованный пакет напрямую:

```bash
npx -y omniroute@latest
```

Можно также установить его глобально:

```bash
npm install -g omniroute
omniroute
```

## Запуск

Запустите OmniRoute в режиме headless-сервера:

```bash
omniroute
```

или:

```bash
npx omniroute
```

Панель управления слушает на:

```text
http://localhost:20128
```

Откройте этот URL в браузере Android. Если вы запускаете клиенты внутри Termux, используйте тот же хост и порт в качестве OpenAI-совместимого base URL.

## Фоновое выполнение

Для простого фонового процесса:

```bash
nohup omniroute > omniroute.log 2>&1 &
```

Чтобы остановить:

```bash
pkill -f omniroute
```

Для автоматического запуска после загрузки устройства установите дополнение Termux:Boot и создайте загрузочный скрипт:

```bash
mkdir -p ~/.termux/boot
cat > ~/.termux/boot/omniroute.sh <<'EOF'
#!/data/data/com.termux/files/usr/bin/sh
cd "$HOME"
nohup omniroute > "$HOME/omniroute.log" 2>&1 &
EOF
chmod +x ~/.termux/boot/omniroute.sh
```

Оптимизация батареи Android может останавливать долго работающие фоновые процессы. Отключите оптимизацию батареи для Termux, если сервер должен оставаться в сети.

## Доступ с других устройств

Найдите IP-адрес телефона в сети WiFi:

```bash
ip addr show wlan0
```

Затем откройте панель управления с другого устройства:

```text
http://PHONE_IP:20128
```

Например:

```text
http://192.168.1.50:20128
```

Держите телефон и клиент в одной доверенной сети. Если вы открываете OmniRoute за пределы телефона, включите API-ключи и аутентификацию панели управления.

## Каталог данных

По умолчанию OmniRoute хранит данные в домашнем каталоге Termux, следуя тому же поведению серверного пути данных, что и на Linux. Чтобы разместить базу данных в явном месте:

```bash
export DATA_DIR="$HOME/.omniroute"
omniroute
```

## Ограничения

- Electron не работает в Termux.
- Нет системного трея и интеграции с рабочим столом.
- Эта конфигурация — только сервер: используйте браузерную панель управления.
- Нативные зависимости могут потребовать локальной компиляции.
- На Android-устройствах с малым объёмом памяти может потребоваться сократить число одновременных запросов.
- Функции MITM/системных сертификатов могут потребовать работы с хранилищем доверия на уровне Android вне Termux.

## Устранение неполадок

### Ошибки сборки better-sqlite3

Установите инструменты сборки Termux:

```bash
pkg install nodejs python build-essential
```

Затем повторите:

```bash
npx -y omniroute@latest
```

### Порт уже занят

Проверьте, что слушает порт по умолчанию:

```bash
ss -ltnp | grep 20128
```

Остановите старый процесс:

```bash
pkill -f omniroute
```

### Панель управления недоступна с другого устройства

Убедитесь, что оба устройства находятся в одной сети WiFi, затем проверьте из Termux:

```bash
curl http://localhost:20128
```

Если локальный доступ работает, а доступ по LAN — нет, проверьте изоляцию точки доступа/WiFi на Android, а также файрвол или VPN-профиль на телефоне.
