# Отслеживание стоимости и расходов

Как OmniRoute оценивает, записывает и отчитывается о стоимости каждого запроса — и почему
цифра на панели управления — это **трекер экономии**, а не счёт.

См. также: [Руководство пользователя](./USER_GUIDE.md) · [Галерея возможностей](./FEATURES.md)

---

## Что это такое (и чем не является)

OmniRoute приписывает каждому завершённому запросу стоимость в USD, перемножая количество
токенов на тарифные ставки модели. Эти цифры питают панель **Costs**, команды
CLI `omniroute cost` / `omniroute usage`, экспорты CSV/JSON и бюджеты на каждый API-ключ.

> **«Стоимость» на панели — это трекер экономии, а не счёт.** OmniRoute никогда не выставляет вам счёт
> — он маршрутизирует ваши запросы к провайдерам, которых вы уже подключили (ваши собственные
> подписки, бесплатные тарифы и API-ключи). «$290 total cost», накопленная целиком на бесплатных
> моделях, означает примерно **$290, которые вы _не_ заплатили** платному API. Цифра — это _оценка_
> того, сколько стоил бы тот же трафик по стандартным прейскурантным ценам, чтобы вы видели,
> где сосредоточено ваше использование и сколько вы экономите маршрутизацией на более дешёвых/бесплатных провайдеров.

Эта трактовка прямо указана в [README](../../README.md) проекта («the dashboard
'cost' is a savings tracker, not a bill»).

Поскольку цифра является оценкой:

- Она зависит от таблицы цен, которая есть у OmniRoute для каждой модели. Модель без записи о цене
  вносит `0` в стоимость (отображается как строка «Legacy / Free» в обозревателе).
- Трафик бесплатных тарифов и подписок всё равно накапливает _оценочную_ стоимость — это
  сумма, которую вы экономите, а не сумма к оплате.

---

## Как оцениваются расходы

### Источник цен

Расходы берутся из таблицы цен, разрешаемой в следующем порядке приоритета
([`src/lib/pricingSync.ts`](../../src/lib/pricingSync.ts)):

1. **Пользовательские переопределения** — цены, заданные вами в панели управления / через `PATCH /api/pricing`.
2. **Синхронизированные внешние цены** — загружаются из публичного
   `model_prices_and_context_window.json` LiteLLM, когда синхронизация включена (хранятся в отдельном
   пространстве имён `pricing_synced`, чтобы никогда не перезаписать ваши переопределения).
3. **Встроенные значения по умолчанию** — поставляются с OmniRoute.

Синхронизация внешних цен — **opt-in**, по умолчанию выключена. Соответствующие переменные окружения
(см. [`.env.example`](../../.env.example)):

| Переменная              | По умолчанию | Назначение                                                       |
| ----------------------- | ------------ | ---------------------------------------------------------------- |
| `PRICING_SYNC_ENABLED`  | `false`      | Включить фоновую синхронизацию цен LiteLLM при запуске.          |
| `PRICING_SYNC_INTERVAL` | `86400`      | Интервал синхронизации в **секундах** (по умолчанию ежедневно).  |
| `PRICING_SYNC_SOURCES`  | `litellm`    | Список источников через запятую (сегодня поддерживается только `litellm`). |

### Формула стоимости

Стоимость вычисляется на каждый запрос из количества токенов и ставок за миллион токенов в
[`src/lib/usage/costCalculator.ts`](../../src/lib/usage/costCalculator.ts)
(`computeCostFromPricing` / `calculateCost`):

- **Входные токены** (минус токены чтения кэша и создания кэша) × ставка `input`.
- **Токены чтения кэша** × ставка `cached` (откат на ставку input).
- **Токены создания кэша** × ставка `cache_creation` (откат на ставку input).
- **Выходные токены** × ставка `output`.
- **Токены рассуждений** × ставка `reasoning` (откат на ставку output).

Все ставки трактуются как USD за 1 000 000 токенов. Уровень обслуживания Codex «fast»/«priority» или
«flex» применяет множитель стоимости (`getCodexFastCostMultiplier`) — например, flex
тарифицируется со скидкой 50% на токены, отображаемой как **экономия flex** на панели.

Имена моделей сначала нормализуются (префиксы пути провайдера, такие как `openai/` или
`accounts/fireworks/models/`, убираются), чтобы исторические строки по-прежнему совпадали с ценой.

### Как записываются расходы

- Стоимость запроса вычисляется после ответа и записывается в режиме fire-and-forget, так что она
  никогда не добавляет задержку клиенту. Потребление общей квоты планируется на следующий
  тик цикла событий через [`src/lib/quota/spendRecorder.ts`](../../src/lib/quota/spendRecorder.ts).
- Расходы по API-ключу буферизуются и сбрасываются пакетами
  [`SpendBatchWriter`](../../src/lib/spend/batchWriter.ts) (по умолчанию интервал сброса 60 с,
  буфер на 1 000 записей). Настраивается через:

  | Переменная                          | По умолчанию | Назначение                         |
  | ----------------------------------- | ------------ | ---------------------------------- |
  | `OMNIROUTE_SPEND_FLUSH_INTERVAL_MS` | `60000`      | Интервал сброса в миллисекундах.   |
  | `OMNIROUTE_SPEND_MAX_BUFFER_SIZE`   | `1000`       | Макс. буферизованных записей до сброса. |

Цифры расходов на панели **не** читаются из сохранённой суммы в долларах по строке — они
пересчитываются на лету из количества токенов и текущей таблицы цен при каждом
выполнении эндпоинта аналитики. Это значит, что исправление неверной цены (и повторная синхронизация)
обновляет исторические оценки расходов задним числом.

## Панель управления: страница Costs

Страница **Costs** находится по адресу `/dashboard/costs`
(`src/app/(dashboard)/dashboard/costs/`).
Её основное представление — вкладка **Cost Overview**
(`src/app/(dashboard)/dashboard/costs/CostOverviewTab.tsx`),
которая загружает всё из `GET /api/usage/analytics`.

Что она показывает:

- **Плитки расходов** — оценочные расходы за _Сегодня (1d)_, _7d_, _30d_ и выбранное
  окно. Селектор диапазона: `7d`, `30d`, `90d`, `all`.
- **Главные метрики** — запросы за окно, активные провайдеры, активные модели, средняя
  стоимость запроса.
- **Cost Explorer** — сортируемая/фильтруемая таблица с группировкой по **провайдеру**, **модели**,
  **API-ключу**, **аккаунту** или **уровню обслуживания**, со стоимостью, запросами, токенами, средней
  стоимостью/запрос и долей от общего %.
- **Использование токенов** — всего / входные / выходные токены и соотношение вход:выход.
- **Эффективность маршрутизации** — число переключений, доля переключений и покрытие запрошенных моделей.
- **Прогноз на месяц** — проецирует расходы к концу месяца по недавнему дневному среднему.
- **Сравнение периодов** — изменение в % между первой и второй половиной окна.
- **Графики** — дневной тренд расходов, доля провайдеров (круговая), топ провайдеров, топ моделей, расходы
  по API-ключу, расходы по аккаунту, недельный паттерн использования и тепловая карта активности.
- **Экспорт** — скачивание текущего окна в **CSV** или **JSON** (кнопки появляются,
  когда есть ненулевые данные о расходах).

Когда нет тарифицируемого трафика, строки отображаются с меткой «Legacy / Free» вместо `$0`,
отражая модель трекера экономии.

### Связанные подстраницы Costs

Область Costs также включает (всё под `/dashboard/costs/`):

- **Pricing** (`/dashboard/costs/pricing`) — просмотр и переопределение цен на модели (рендерит
  общую вкладку Pricing).
- **Budget** (`/dashboard/costs/budget`) — установка лимитов расходов по областям (рендерит общую
  вкладку Budget).
- **Quota Share** (`/dashboard/costs/quota-share`) — пулы общей квоты и представления скорости
  расходования.

## Эндпоинты API

Все они требуют управленческой авторизации (loopback/JWT, через `requireManagementAuth`), если не
указано иное.

### Аналитика использования и расходов

| Метод  | Эндпоинт                 | Назначение                                                                                                                                                |
| ------ | ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `GET`  | `/api/usage/analytics`   | Полная аналитика расходов/использования: сводка, дневной тренд, по провайдеру/модели/API-ключу/аккаунту/уровню. Query: `range`, `startDate`, `endDate`, `apiKeyIds`, `presets`. |
| `GET`  | `/api/usage/utilization` | Утилизация квоты по провайдерам во времени. Query: `range` (`1h`/`24h`/`7d`/`30d`), `provider`.                                                           |
| `GET`  | `/api/usage/history`     | Сырые строки истории использования.                                                                                                                       |
| `GET`  | `/api/usage/call-logs`   | Журналы вызовов по запросам (модель, токены, стоимость, задержка, статус).                                                                                |
| `GET`  | `/api/usage/quota`       | Статус квоты провайдеров.                                                                                                                                 |
| `GET`  | `/api/usage/proxy-logs`  | Журналы прокси-запросов.                                                                                                                                  |

### Бюджеты

| Метод  | Эндпоинт                 | Назначение                                                                     |
| ------ | ------------------------ | ------------------------------------------------------------------------------ |
| `GET`  | `/api/usage/budget`      | Сводка расходов + проверка бюджета для одного API-ключа (обязателен query `apiKeyId`). |
| `POST` | `/api/usage/budget`      | Установить дневные/недельные/месячные лимиты USD + порог предупреждения для API-ключа. |
| `GET`  | `/api/usage/budget/bulk` | Массовые сводки бюджетов по API-ключам.                                        |

> API бюджетов работает в разрезе **API-ключа** (`apiKeyId`). Лимиты, возвращаемые
> `GET /api/usage/budget`, включают `dailyLimitUsd`, `weeklyLimitUsd`, `monthlyLimitUsd`,
> `warningThreshold` и накопленные итоги (`totalCostToday`, `totalCostMonth`, …).

### Цены

| Метод    | Эндпоинт                | Назначение                                                                                      |
| -------- | ----------------------- | ----------------------------------------------------------------------------------------------- |
| `GET`    | `/api/pricing`          | Текущие объединённые цены (пользовательские + синхронизированные + по умолчанию). `?includeSources=1` — источник каждой записи. |
| `PATCH`  | `/api/pricing`          | Переопределить цены для `{ provider: { model: { input, output, cached, … } } }`.                |
| `DELETE` | `/api/pricing`          | Сбросить цены к значениям по умолчанию (опционально с областью `?provider=&model=`).            |
| `GET`    | `/api/pricing/defaults` | Показать резервные ставки за 1M по умолчанию.                                                   |
| `GET`    | `/api/pricing/models`   | Цены по моделям.                                                                                |
| `POST`   | `/api/pricing/sync`     | Запустить ручную синхронизацию из внешних источников (LiteLLM).                                 |
| `GET`    | `/api/pricing/sync`     | Текущий статус синхронизации.                                                                   |
| `DELETE` | `/api/pricing/sync`     | Очистить все синхронизированные данные о ценах.                                                 |

### Другие эндпоинты, связанные с расходами

| Метод | Эндпоинт                      | Назначение                                                               |
| ----- | ----------------------------- | ------------------------------------------------------------------------ |
| `GET` | `/api/free-tier/summary`      | Итоги токенов бесплатных моделей, использовано в этом месяце и оставшаяся бесплатная квота. |
| `GET` | `/api/quota/pools/[id]/usage` | Использование пула общей квоты.                                          |

## CLI

CLI OmniRoute предоставляет команды расходов, использования и цен (регистрируются в
[`bin/cli/commands/registry.mjs`](../../bin/cli/commands/registry.mjs)).

### `omniroute cost`

Отчёт о расходах, агрегированный из `/api/usage/analytics`.

```bash
omniroute cost                          # последние 30 дней, группировка по провайдеру
omniroute cost --period 7d              # последние 7 дней
omniroute cost --group-by model         # группировать по provider | model | combo | api-key | day
omniroute cost --since 2026-06-01 --until 2026-06-13
omniroute cost --api-key <key> --limit 50
```

Колонки: группа, запросы, входные/выходные токены, стоимость (USD) и % от общего. Итоговая строка
печатается в конце (подавляется флагами `--quiet` или `--output json`).

### `omniroute usage`

```bash
omniroute usage analytics --period 30d [--provider <id>]   # сводка расходов по провайдерам
omniroute usage logs [--limit 100] [--follow] [--api-key <k>] [--search <q>]
omniroute usage quota [--provider <id>] [--check]
omniroute usage utilization [--api-key <k>]
omniroute usage history [--limit 100]
omniroute usage proxy-logs [--limit 100]

# Бюджеты
omniroute usage budget list
omniroute usage budget get [scope]
omniroute usage budget set <amount> [--scope global] [--period monthly]
omniroute usage budget reset [scope]
```

### `omniroute pricing`

```bash
omniroute pricing list [--provider <p>] [--model <m>] [--limit 200]
omniroute pricing get <model>
omniroute pricing sync [--provider <p>] [--force]   # POST /api/pricing/sync
omniroute pricing diff [--model <m>]
omniroute pricing defaults show
omniroute pricing defaults set [--input <p>] [--output <p>] [--cache-read <p>] [--cache-write <p>]
```

> `pricing defaults show` читает `GET /api/pricing/defaults`. Чтобы редактировать цены отдельных моделей,
> используйте страницу **Pricing** в панели управления или `PATCH /api/pricing`.

## Устранение неполадок

- **Все расходы показываются как $0 / «Legacy / Free».** У используемых моделей нет записи о цене.
  Включите внешнюю синхронизацию (`PRICING_SYNC_ENABLED=true`) и выполните `omniroute pricing sync` или
  задайте цены вручную через страницу Pricing / `PATCH /api/pricing`.
- **Историческая модель оценена неверно.** Исправьте цену (переопределением или повторной синхронизацией) — стоимость
  пересчитывается из количества токенов при каждом чтении аналитики, поэтому оценки обновляются задним числом.
- **Расходы отстают от реального времени.** Расходы по ключу обрабатываются пакетами; уменьшите
  `OMNIROUTE_SPEND_FLUSH_INTERVAL_MS`, если нужны более свежие цифры.
