# Руководство по Progressive Web App (PWA)

OmniRoute поставляется как полностью устанавливаемое Progressive Web App. Когда вы открываете панель управления из любого мобильного браузера — Android (Chrome) или iOS (Safari) — вы можете выбрать «Добавить на главный экран» и получить опыт, как у нативного приложения, без магазина приложений.

## Что такое PWA?

Progressive Web App превращает веб-панель OmniRoute в нечто, что выглядит и ощущается как нативное мобильное приложение. После установки оно:

- Запускается с главного экрана с собственной иконкой
- Открывается в полноэкранном режиме — без адресной строки браузера и вкладок
- Работает офлайн со специальной страницей о состоянии связи
- Кэширует статические ресурсы для более быстрой загрузки
- Поддерживает книжную и альбомную ориентации

## Установка

### Android (Chrome)

1. Откройте панель управления OmniRoute в Chrome: `http://YOUR_IP:20128`
2. Chrome автоматически покажет баннер **«Add OmniRoute to Home screen»**, или:
   - Нажмите меню **⋮** (три точки) → **«Add to Home screen»** или **«Install app»**
3. Подтвердите запрос
4. OmniRoute появится на главном экране как отдельное приложение

### iOS (Safari)

1. Откройте панель управления OmniRoute в Safari: `http://YOUR_IP:20128`
2. Нажмите кнопку **Поделиться** (квадрат со стрелкой)
3. Прокрутите вниз и нажмите **«Add to Home Screen»**
4. Задайте имя (по умолчанию «OmniRoute») и нажмите **Add**
5. OmniRoute появится на главном экране с иконкой приложения

### Десктоп (Chrome / Edge)

1. Откройте панель управления OmniRoute
2. Нажмите **иконку установки** в адресной строке (или ⋮ → «Install OmniRoute...»)
3. Подтвердите запрос
4. OmniRoute откроется как отдельное окно — без вкладок и адресной строки

## Возможности

### Полноэкранный режим

Манифест настроен с `display: "fullscreen"`, то есть установленное приложение использует весь экран — без интерфейса браузера и перекрытия строки состояния. Благодаря этому панель управления ощущается по-настоящему нативной.

### Поддержка офлайн-режима

OmniRoute включает service worker (`sw.js`), обеспечивающий интеллектуальное кэширование:

| Тип ресурса                                             | Стратегия                          | Поведение                                                                    |
| ------------------------------------------------------- | ---------------------------------- | ---------------------------------------------------------------------------- |
| **Каркас приложения**                                   | Cache-first                        | `/`, `/offline`, манифест и иконки предкэшируются при установке              |
| **Статические ресурсы** (CSS, JS, изображения, шрифты)  | Network-first с откатом в кэш      | Запрашивает свежее из сети; откатывается в кэш при офлайне                   |
| **Бандлы Next.js** (`/_next/`)                          | Network-first с обновлением кэша   | Запрашивает из сети и обновляет кэш; отдаёт кэшированную версию при офлайне  |
| **Навигационные запросы**                               | Network-only с офлайн-откатом      | Всегда запрашивает из сети; показывает страницу `/offline`, если сеть недоступна |
| **API-маршруты** (`/api/`, `/a2a`, `/dashboard/endpoint`) | Обход (никогда не кэшируются)    | Всегда идут напрямую на сервер — service worker их не перехватывает          |

### Офлайн-страница

Когда сеть недоступна и пользователь переходит на новую страницу, service worker отдаёт специальную страницу `/offline`, которая:

- Показывает понятное сообщение **«Connectivity Issue»**
- Отображает живой **индикатор онлайн/офлайн-статуса**, обновляющийся в реальном времени
- Предоставляет кнопку **«Retry Connection»** для перезагрузки при восстановлении связи
- Ссылается на **страницу статуса** для диагностики

### Иконки приложения

OmniRoute предоставляет иконки, оптимизированные для каждой платформы:

| Файл                   | Размер           | Используется                            |
| ---------------------- | ---------------- | --------------------------------------- |
| `icon-512.png`         | 512×512          | Запрос установки Android, заставка      |
| `apple-touch-icon.png` | 180×180          | Иконка главного экрана iOS              |
| `icon-192.svg`         | 192×192 (вектор) | Адаптивная иконка Android               |
| `apple-touch-icon.svg` | 180×180 (вектор) | Резервный вариант для Apple             |
| `favicon.svg`          | Вектор           | Вкладки браузера                        |
| `favicon.ico`          | Мультиразмер     | Устаревшие браузеры                     |

### Автоматическая регистрация

Service worker регистрируется автоматически через компонент `<PwaRegister />` в корневом макете. Действия пользователя не требуются — приложение становится устанавливаемым, как только браузер обнаружит корректный манифест и service worker.

## Техническая архитектура

### Манифест веб-приложения (`manifest.webmanifest`)

Генерируется Next.js через `src/app/manifest.ts`:

```json
{
  "name": "OmniRoute",
  "short_name": "OmniRoute",
  "description": "OmniRoute is an AI gateway for multi-provider LLMs. One endpoint for all your AI providers.",
  "start_url": "/",
  "scope": "/",
  "display": "fullscreen",
  "orientation": "any",
  "background_color": "#0b0f1a",
  "theme_color": "#0b0f1a",
  "icons": [
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png", "purpose": "any maskable" },
    { "src": "/apple-touch-icon.png", "sizes": "180x180", "type": "image/png" }
  ]
}
```

### Service Worker (`public/sw.js`)

«Ванильный» service worker (без зависимостей от фреймворков) с:

- **Фазой установки**: предкэширует каркас приложения (корень, офлайн-страницу, манифест, иконки)
- **Фазой активации**: очищает старые версии кэша и захватывает всех клиентов
- **Фазой fetch**: интеллектуальная маршрутизация по типу запроса (навигация, статический ресурс, API)
- **Версионированием кэша**: `omniroute-pwa-v2` — увеличьте его, чтобы принудительно сбросить кэш при обновлении

### Метаданные макета (`src/app/layout.tsx`)

Корневой макет предоставляет все метатеги, необходимые для соответствия PWA:

- ссылка `manifest` на `/manifest.webmanifest`
- `apple-web-app-capable: true` для автономного режима iOS
- `apple-web-app-status-bar-style: black-translucent`
- `mobile-web-app-capable: yes` для Android Chrome
- `theme-color: #0b0f1a`
- `viewport-fit: cover` для отрисовки от края до края

### Компонент: `PwaRegister`

Расположен в `src/shared/components/PwaRegister.tsx`, этот клиентский компонент:

1. Выполняется при монтировании (только на клиенте)
2. Проверяет поддержку `serviceWorker` в браузере
3. Регистрирует `/sw.js` молча (ошибки поглощаются, чтобы не блокировать приложение)
4. Ничего не рендерит (`return null`) — это компонент только с побочным эффектом

## Использование с Termux (Android)

Когда OmniRoute запущен на Android через Termux, PWA работает без проблем:

1. Запустите OmniRoute в Termux: `npx omniroute`
2. Откройте Chrome на том же телефоне: `http://localhost:20128`
3. Установите PWA через «Add to Home Screen»
4. PWA подключается к локальному серверу Termux — всё работает на устройстве

Эта комбинация означает, что ваш Android-телефон одновременно и **сервер** (Termux), и **клиент** (PWA) — полностью автономный ИИ-шлюз.

## Использование с других устройств

Установите PWA на любом устройстве, у которого есть браузерный доступ к вашему серверу OmniRoute:

- **Другой телефон/планшет**: перейдите на `http://PHONE_IP:20128` и установите PWA
- **Ноутбук**: откройте Chrome/Edge и установите как десктопное PWA
- **Smart TV с браузером**: откройте панель управления в полноэкранном режиме

## Настройка

### Имя экземпляра

Заголовок PWA учитывает настройку **Instance Name** из `Dashboard → Settings`. Если вы переименуете экземпляр в «My AI Gateway», установленное PWA покажет это имя.

### Пользовательский favicon

Если вы загрузите собственный favicon через `Dashboard → Settings`, иконка PWA на десктопе отразит пользовательскую иконку. Иконки главного экрана на мобильных используют готовые файлы `icon-512.png` и `apple-touch-icon.png`.

## Ограничения

- **Нет push-уведомлений** — service worker не реализует Push API. Уведомления обрабатываются приложением Electron.
- **Нет фоновой синхронизации** — офлайн-действия не ставятся в очередь для повторного воспроизведения. PWA — это в первую очередь просмотрщик панели управления.
- **Ограничения iOS** — Safari на iOS поддерживает не все возможности PWA (например, запросы на установку ручные, а фоновые service worker ограничены).
- **Размер кэша** — service worker кэширует только статические ресурсы. Большая полезная нагрузка ответов маршрутов `/api/` никогда не кэшируется.
- **Пользовательские иконки на мобильных** — смена favicon в настройках не обновляет иконку главного экрана на мобильных (для этого нужно пересоздавать иконки PWA).
