# Рейтинги бесплатных провайдеров (Arena ELO)

> **TL;DR**: OmniRoute ранжирует своих **бесплатных** провайдеров по качеству моделей, используя **оценки ELO от Arena AI
> (в стиле LMArena)**. Откройте страницу **Free Provider Rankings** в
> панели управления, чтобы увидеть, какие бесплатные провайдеры предлагают самые сильные модели для вашей задачи —
> в целом или с фильтрацией по категории (код, ревью, документация, отладка).

---

## Что это такое

OmniRoute объединяет 160+ провайдеров, многие из которых предоставляют **бесплатный тариф** (без авторизации,
OAuth с бесплатным тарифом или API-ключ с бесплатным тарифом — см.
[руководство по бесплатным тарифам](../getting-started/FREE-TIERS-GUIDE.md) и полный
[каталог бесплатных тарифов](../reference/FREE_TIERS.md)). Проблема в том, что бесплатные провайдеры сильно
различаются по качеству моделей. Провайдер без авторизации, обслуживающий фронтирную модель, гораздо полезнее,
чем тот, кто обслуживает небольшую устаревшую модель.

**Free Provider Rankings** отвечает на вопрос «**какой бесплатный провайдер даст мне лучшую модель?**»,
связывая каталог каждого бесплатного провайдера с **краудсорсинговыми оценками качества** из
**рейтинга Arena AI** (ELO человеческих предпочтений — та же идея, что лежит в основе
арены чатботов LMArena). Затем провайдеры ранжируются по силе их **лучшей бесплатной модели**.

Рейтинг вычисляется из трёх реальных источников:

1. Списки бесплатных провайдеров — `NOAUTH_PROVIDERS`, плюс записи `OAUTH_PROVIDERS` /
   `APIKEY_PROVIDERS` с флагом `hasFree`
   (`src/shared/constants/providers.ts`).
2. Каталог моделей каждого провайдера из реестра провайдеров
   (`open-sse/config/providerRegistry.ts`).
3. Производные от ELO оценки соответствия задаче, хранящиеся в таблице БД `model_intelligence`,
   записываемые движком синхронизации Arena ELO (`src/lib/arenaEloSync.ts`).

Логика объединения находится в `src/lib/freeProviderRankings.ts`.

## Как получить доступ

### Страница в панели управления

Откройте панель управления и перейдите в **Costs → Free Provider Rankings** или напрямую по адресу:

```
/dashboard/free-provider-rankings
```

Страница (`src/app/(dashboard)/dashboard/free-provider-rankings/page.tsx`) показывает:

- **Пьедестал топ-3** (🥇 🥈 🥉) лучших бесплатных провайдеров.
- Полную **таблицу рейтинга** с колонками: **Rank**, **Provider**, **Top Model**,
  **Score**, **Avg Score**, **Models**, **Type**.
- **Кнопки фильтра категорий**: _All Categories_, _Default_, _Coding_, _Review_,
  _Documentation_, _Debugging_.

Значок **Type** каждого провайдера показывает, как именно он бесплатный:

| Значок   | Значение                                       |
| -------- | ---------------------------------------------- |
| `NOAUTH` | Всегда бесплатный, учётные данные не нужны     |
| `OAUTH`  | OAuth-провайдер с бесплатным тарифом (`hasFree`) |
| `APIKEY` | Провайдер с API-ключом и бесплатным тарифом (`hasFree`) |

Оценки показаны как человекочитаемые метки (напр. _Elite_, _Excellent_, _Very Good_,
_Good_, _Average_), а не сырые числа, потому что нижележащее значение — это относительное
качество в рейтинге, а не процент.

### Эндпоинт API

Страница обслуживается публичным эндпоинтом чтения
(`src/app/api/free-provider-rankings/route.ts`):

```
GET /api/free-provider-rankings
GET /api/free-provider-rankings?category=coding
GET /api/free-provider-rankings?category=coding&limit=20
```

Параметры запроса (валидируются Zod):

| Параметр   | Тип    | По умолчанию | Примечания                                                                                         |
| ---------- | ------ | ------------ | -------------------------------------------------------------------------------------------------- |
| `category` | string | (нет)        | Одно из `default`, `coding`, `review`, `documentation`, `debugging`. Опустите для общего рейтинга. |
| `limit`    | number | `50`         | Ограничивается диапазоном `1–100`.                                                                 |

Форма ответа:

```json
{
  "rankings": [
    {
      "id": "<provider-id>",
      "name": "<provider name>",
      "icon": "<icon>",
      "color": "<hex color>",
      "textIcon": "<short label>",
      "category": "noauth | oauth | apikey",
      "topModel": {
        "modelId": "<registry model id>",
        "modelName": "<model display name>",
        "score": 0.0,
        "eloRaw": 0,
        "confidence": "high | medium | low",
        "category": "<task category>"
      },
      "averageScore": 0.0,
      "modelCount": 0
    }
  ]
}
```

`eloRaw` — исходное значение Arena ELO; `score` — нормализованное значение соответствия задаче
(см. ниже). Провайдеры без оценённых моделей исключаются из ответа.

## Как работают оценки

### Источник: рейтинг Arena AI

Движок синхронизации Arena ELO (`src/lib/arenaEloSync.ts`) запрашивает два рейтинга — `text`
и `code` — из API рейтинга Arena AI
(`https://api.wulong.dev/arena-ai-leaderboards/v1/leaderboard`). Каждая запись рейтинга
содержит имя модели, вендора, `score` ELO, доверительный интервал и число голосов.

Категории рейтинга отображаются на категории задач OmniRoute:

| Рейтинг Arena | Категории задач OmniRoute                         |
| ------------- | ------------------------------------------------- |
| `text`        | `default`, `review`, `documentation`, `debugging` |
| `code`        | `coding`                                          |

### Нормализация (оценка соответствия задаче)

Сырые оценки ELO нормализуются внутри каждого рейтинга в **значение соответствия задаче в диапазоне `[0.4, 0.98]`**:

```
taskFit = 0.4 + 0.58 * ((elo - minElo) / (maxElo - minElo))
```

Оценка никогда не достигает `0` или `1`, оставляя запас для пользовательских переопределений. Это
поле `score`, которое вы видите в ответе API, и метка на панели управления.

### Уверенность

Каждая запись получает уровень уверенности на основе числа голосов Arena:

| Уверенность | Голоса  |
| ----------- | ------- |
| `high`      | ≥ 5 000 |
| `medium`    | ≥ 1 000 |
| `low`       | < 1 000 |

### Хранение и свежесть

Нормализованные записи записываются в таблицу БД `model_intelligence` с
`source = "arena_elo"` (`src/lib/db/modelIntelligence.ts`). Записи **истекают через
7 дней**, поэтому провайдер, который перестаёт синхронизироваться, со временем выпадает из рейтинга,
а не обслуживает устаревшие данные.

Синхронизация работает **по умолчанию**:

- Она запускается один раз при старте сервера, а затем по периодическому таймеру
  (`src/lib/arenaEloSync.ts`, подключается из `src/server-init.ts`).
- Она **неблокирующая и никогда не фатальна** — если запрос к апстриму не удался, OmniRoute продолжает
  работать, а рейтинги просто показывают последние корректные данные (или пустое состояние).

Две переменные окружения управляют этим (задокументированы в
[`docs/reference/ENVIRONMENT.md`](../reference/ENVIRONMENT.md)):

| Переменная                | По умолчанию  | Назначение                                               |
| ------------------------- | ------------- | -------------------------------------------------------- |
| `ARENA_ELO_SYNC_ENABLED`  | `true`        | Установите `false`, чтобы отказаться от исходящей синхронизации. |
| `ARENA_ELO_SYNC_INTERVAL` | `86400` (24ч) | Интервал синхронизации в секундах.                       |

### Ручная синхронизация / статус / очистка

Для операторов аутентифицированный управленческий эндпоинт предоставляет ручное управление
(`src/app/api/intelligence/sync/route.ts` — требует управленческой авторизации):

```
GET    /api/intelligence/sync          # текущий статус синхронизации (enabled, lastSync, nextSync, intervalMs)
POST   /api/intelligence/sync          # запустить синхронизацию вручную; тело: { "dryRun": true } для предпросмотра без записи
DELETE /api/intelligence/sync          # очистить все синхронизированные записи arena_elo
```

Если страница рейтингов пуста, ручной `POST /api/intelligence/sync` (или просто
перезапуск сервера) заполнит её заново.

### Сопоставление моделей с рейтингом

ID моделей реестра и имена моделей Arena не всегда совпадают точно. Рейтинг использует
гибкое сопоставление (`findMatchingIntelligence` в `src/lib/freeProviderRankings.ts`):

1. Точное совпадение по нормализованному ID модели.
2. Совпадение после удаления хвостового суффикса версии (напр. `kimi-k2.6` → `kimi-k2`).
3. Совпадение по префиксу (имя модели из рейтинга является префиксом ID реестра).

На стороне синхронизации известные префиксы вендоров (`anthropic/`, `openai/`, `google/`, …)
убираются, а небольшая карта алиасов разворачивает канонические имена в варианты, используемые OmniRoute
внутри, так что модели остаются находимыми под любым именем.

### Как ранжируется провайдер

Для каждого бесплатного провайдера движок оценивает каждую модель в его каталоге, затем:

- **Top Model** = модель провайдера с наивысшей оценкой.
- **Avg Score** = средняя оценка по всем оценённым моделям этого провайдера.
- **Models** = сколько моделей провайдера получили оценку Arena.

Провайдеры сортируются сначала по **оценке лучшей модели**, затем по средней оценке. Это вознаграждает
провайдера, предлагающего хотя бы одну сильную бесплатную модель.

## Как использовать это для выбора бесплатных провайдеров

1. **Выберите правильную категорию.** Используйте фильтр **Coding** для агентных/кодовых нагрузок или
   оставьте **All Categories** / **Default** для общего чата. Один и тот же провайдер может
   ранжироваться по-разному в разных категориях, потому что его лучшая модель отличается в разных рейтингах.
2. **Ориентируйтесь на пьедестал для разовых настроек.** Если вы хотите подключить только одного-двух бесплатных
   провайдеров, начните с самых высокоранговых для вашей категории.
3. **Проверьте значок Type.** Провайдеры `NOAUTH` подключаются быстрее всего (без
   учётных данных). Бесплатные тарифы `OAUTH` / `APIKEY` требуют быстрой регистрации, но часто предоставляют
   более сильные модели. Шаги подключения см. в [руководстве по бесплатным тарифам](../getting-started/FREE-TIERS-GUIDE.md).
4. **Подключите несколько и позвольте Auto-Combo решать.** Те же данные Arena ELO, что питают
   эту страницу, также питают **фактор соответствия задаче** движка оценки Auto-Combo
   (`open-sse/services/autoCombo/taskFitness.ts`, порядок разрешения
   `user_override → arena_elo → models_dev_tier → static table`). Поэтому после подключения
   лучших бесплатных провайдеров маршрутизация с `model: "auto"` (напр. `auto/coding`) будет
   автоматически предпочитать более качественные бесплатные модели на каждый запрос. См.
   [Auto-Combo](../routing/AUTO-COMBO.md) для полного 9-факторного скоринга.
